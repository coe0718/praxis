# Crypto

> AES-256-GCM encryption at rest and secret zeroization

## Overview

The `crypto` module provides encryption for sensitive data files stored on disk. It uses AES-256-GCM authenticated encryption with a 32-byte master key stored at `master.key` in the data directory. The module is designed to be opt-in: installations that haven't run `praxis init` operate in plaintext mode, and encryption is applied transparently when the master key exists.

The encryption format is deliberately text-friendly: encrypted blobs are stored as `PRAXISENC1:` followed by hex-encoded nonce and ciphertext. This keeps encrypted files diff-able for version control while remaining opaque to casual inspection.

A companion `zeroize` submodule provides a `SecretString` wrapper that securely overwrites memory on drop, preventing sensitive data from lingering in process memory.

## Architecture

### Key Management

- The 32-byte master key is stored at `data_dir/master.key`.
- Generated on `praxis init` using `Aes256Gcm::generate_key(OsRng)`.
- On Unix, the key file is set to mode `0o600` (owner read/write only).
- A warning is logged if the key file has insecure permissions.

### Encryption Format

```
PRAXISENC1:<hex(nonce 12B)><hex(ciphertext + 16B auth tag)>
```

- Nonce: 12 bytes (96 bits), randomly generated per encryption call.
- Ciphertext includes the 16-byte GCM authentication tag.
- A fresh nonce is used for every encryption, so identical plaintexts produce different ciphertexts.

### Key Types

- **`SecretString`** — A `String` wrapper that zeroizes its contents on drop using the `zeroize` crate. Provides `expose()` for read access and `into_inner()` for ownership transfer.

### Key Functions

| Function | Purpose |
|---|---|
| `load_or_generate_key(path)` | Load existing key or generate and persist a new one |
| `encrypt(key, plaintext)` | Encrypt bytes → `PRAXISENC1:...` string |
| `decrypt(key, encoded)` | Decrypt a `PRAXISENC1:...` blob → plaintext bytes |
| `is_encrypted(raw)` | Check if a string has the encryption magic prefix |
| `maybe_encrypt(path, raw)` | Encrypt if master key exists, else pass through |
| `maybe_decrypt(path, raw)` | Decrypt if `PRAXISENC1:` prefix found, else pass through |
| `zeroize_string(s)` | Overwrite a `String` with zeros in place |
| `zeroize_bytes(b)` | Overwrite a byte slice with zeros in place |

## Public API

```rust
// Key management
pub fn load_or_generate_key(path: &Path) -> Result<[u8; 32]>

// Encryption/decryption
pub fn encrypt(key: &[u8; 32], plaintext: &[u8]) -> Result<String>
pub fn decrypt(key: &[u8; 32], encoded: &str) -> Result<Vec<u8>>
pub fn is_encrypted(raw: &str) -> bool

// Convenience helpers for file stores
pub fn maybe_encrypt(path: &Path, raw: &str) -> Result<String>
pub fn maybe_decrypt(path: &Path, raw: &str) -> Result<String>

// Zeroization (in crypto::zeroize submodule)
pub fn zeroize_string(s: &mut String)
pub fn zeroize_bytes(b: &mut [u8])

// SecretString wrapper
SecretString::new(s) -> Self
SecretString::expose(&self) -> &str
SecretString::into_inner(self) -> String
```

## Configuration

No `praxis.toml` fields. Encryption is controlled by the presence of `master.key`:
- **Key exists** → Files are encrypted on write and decrypted on read.
- **Key absent** → Files are stored in plaintext (backward-compatible).

The master key file is generated by `praxis init` and should never be committed to version control (excluded by the generated `.gitignore`).

## Usage

### Automatic

Encryption is applied transparently by file stores that handle sensitive data (e.g., OAuth tokens, API credentials). The `maybe_encrypt`/`maybe_decrypt` helpers check for the master key and the encryption prefix, so code doesn't need to know whether encryption is active.

### Manual

```rust
// Encrypt a secret
let key = crypto::load_or_generate_key(&paths.data_dir.join("master.key"))?;
let encrypted = crypto::encrypt(&key, b"my secret value")?;

// Check if content is encrypted
if crypto::is_encrypted(&raw) {
    let plaintext = crypto::decrypt(&key, &raw)?;
}

// Use SecretString for sensitive values in memory
let secret = SecretString::new("api-key-here");
// ... use secret.expose() ...
// On drop, the string's memory is zeroized
```
